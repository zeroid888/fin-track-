# Fin track — контекст проекта

## Что это

Персональный финансовый трекер для одного пользователя (Daniel). Один файл `index.html` со встроенными CSS и JS — никаких сборок, никаких зависимостей. Всё работает локально в браузере, данные хранятся в `localStorage` под ключом `myfinance_v2`. Открывается с телефона как обычная веб-страница.

## Кто пользователь

Daniel, по его собственным словам — «финансово неграмотен». Цели:
- Видеть все доходы и расходы в одном месте.
- Понимать, сколько можно тратить в день, чтобы влезать в месяц и гасить кредиты.
- Получить «средний чек жизни», чтобы понимать, где можно подкорректировать.
- Расти в доходах.

Поэтому интерфейс должен быть на русском, без жаргона, с подсказками и без скрытых автоматических действий с деньгами.

## Финансовый профиль (на момент написания)

- **Доходы:**
  - Зарплата 175 000₽, два раза в месяц (5 и 20 числа) — итого ~350 000₽/мес.
  - Bitzone — пассивный, **в USDT**, ~30₮/день, льётся на счёт «BitZone's».
  - От тимлида — ежедневно, **в USD**, $30–40/день.
- **Балансы:** рублёвый осевой счёт + крипто-накопительный «BitZone's» (USDT).
- **Кредиты / обязательные расходы:** аренда списывается с BitZone's раз в месяц 2-го числа (~987₮).
- **Привычки:** разная частота — спорт каждый день, массаж раз в неделю, шоппинг 2 раза в месяц.

## Технический стек и архитектура

- **Один файл:** `/sessions/jolly-practical-fermat/mnt/Fin track/index.html` (~5300 строк после всех правок).
- **Без сборки и без зависимостей** — vanilla HTML/CSS/JS.
- **Хранилище:** `localStorage['myfinance_v2']`.
- **Локаль:** ru-RU, даты в ISO (YYYY-MM-DD).
- **Структура `state`:**
  ```js
  {
    incomes: [{ id, name, amount, currency:'RUB|USD|USDT', type, freq, payDays, balanceId? }],
    expenses: [{ id, name, amount, type, cat, payDay?, endDate?, balanceId?, balanceDeductPerMonth?, deductionLog:[], skippedMonths:[] }],
    transactions: [{ id, date, name, amount, cat, type:'income|expense', incomeId?, balanceId?, nativeAmount?, nativeCurrency? }],
    credits: [{ id, name, monthly, total, term, termUnit, freq, payDay?, payDays?, anchorDate? }],
    goals: [{ id, name, emoji, target, saved, monthly, deadline }],
    habits: [{ id, name, emoji, cost, freq:'daily|weekdays|weekly|biweekly|semimonthly|monthly', cat, active }],
    balances: [{ id, name, emoji, amount, currency }],
    rates: { USD: 90, USDT: 90 },
    lastImport: null
  }
  ```

## Соглашения, которые нельзя нарушать

1. **Все суммы в `transactions[].amount` хранятся в ₽.** Если транзакция была в USDT/USD, оригинал кладётся в `nativeAmount` + `nativeCurrency`, в `amount` — конвертация в ₽.
2. **Никогда не удалять и не переименовывать поля `state` без миграции в `load()`** — сохранённые данные пользователя не должны теряться.
3. **Любое движение по балансу — только с подтверждения пользователя.** Никаких silent-deductions. Если по графику пора списать аренду — показать модалку с выбором «списать / уже сам оплатил / пропустить месяц».
4. **Конвертация валют — только через `toRub(amount, cur)` и `fromRub(rubAmount, cur)`** с курсами из `state.rates`.
5. **Цвета:** `--accent` (зелёный) — доход/норма, `--accent2` (красный) — перебор/долг, `#ff9f4f` (оранжевый) — предупреждение.
6. **Тон сообщений:** прямой, по делу, без «возможно» и «может быть». Если плохо — сказать «плохо», а не «обратите внимание».

## Если нужно стартовать с нуля на новом устройстве

1. Скопируй файл `index.html` (или открой его прямо).
2. Заведи балансы: рублёвый счёт + BitZone's (USDT).
3. Заведи доходы: ЗП 5/20, Bitzone (USDT, ежедневно, привязка к BitZone's), тимлид (USD, ежедневно).
4. Заведи аренду в «Расходы»: сумма в ₽, payDay=2, привяжи к BitZone's.
5. Заведи привычки с правильной частотой (спорт=daily, массаж=weekly и т.д.).
6. Каждый день нажимай «＋ Зафиксировать сегодня» у Bitzone и тимлида — балансы обновятся, доход за месяц подтянется.
7. В начале месяца модалка предложит подтвердить списание аренды.
