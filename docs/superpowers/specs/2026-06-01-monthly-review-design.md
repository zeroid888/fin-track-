# Итоги месяца — дизайн

Дата: 2026-06-01. Файл: `index.html` (один файл, vanilla, без сборки).

## Зачем

Месяц закрывается — пользователь не получает никакого ретро-итога («как прошло, что улучшить»). В коде есть только прогноз вперёд (month-end forecast), ретроспективы нет. Добавляем разбор закрытого месяца + живой просмотр текущего.

## Что делаем

1. **Вкладка «Итоги»** (10-й пункт нав): сверху текущий месяц вживую (числа на лету, помечены «идёт», с прикидкой темпа к концу месяца), ниже — закрытые месяцы из архива (замороженные числа).
2. **Авто-модалка 1-го числа**: при первом открытии в новом месяце трекер сам показывает разбор прошлого месяца. Один раз, потом доступен в архиве.
3. Выводы «что улучшить» — **правилами в коде** (без ИИ): офлайн, бесплатно, без ключа в публичном репо. Для арифметики порогов этого достаточно.

## Данные (`state`)

Два новых поля. Оба → в `ALLOWED_KEYS` (строка ~2563) + миграция в `load()`.

- `monthlyReviews` — карта `'YYYY-MM' → {объект разбора}`. **Числа закрытого месяца замораживаются** в момент расчёта: правка старой транзакции не меняет уже показанный итог, архив не пересчитывается.
- `lastReviewSeen` — `'YYYY-MM' | null`, последний показанный в модалке месяц. Защита от повторного всплытия.

Дефолты в литерале `state`: `monthlyReviews: {}`, `lastReviewSeen: null`.
Миграция: `if (!state.monthlyReviews || typeof !== 'object') = {}`; `lastReviewSeen` — строка `YYYY-MM` или null.

## Расчёт — `computeMonthlyReview(ym, opts)`

Чистая функция, возвращает объект. `opts.live=true` для текущего месяца (числа не замораживаются, добавляется прогноз темпа).

- **Транзакции месяца:** `t.date.slice(0,7)===ym`, **исключая** переводы `isTransferTx(t)` (это и есть незакрытый P0-3 — здесь делаем сразу правильно).
- **Доход факт** = сумма `type==='income'`; **доход план** = `incomeScheduleForMonth(inc, y, m)` по всем `state.incomes` за этот месяц (честный план именно за тот месяц, а не «на сегодня»).
- **Расход факт** = сумма `type!=='income'`; **расход план** = `totalExpenses()+totalCredits()+totalHabitsMonthly()` (текущий конфиг — приближение, помечаем как «план»).
- **Категории:** группировка факт-расхода по `cat` → `[{cat, sum, count}]`, сверху по сумме.
- **Нетто** = доход факт − расход факт.
- **Средний день** = расход факт / дней в месяце (`new Date(y,m,0).getDate()`, не `/30`). Для live: / прошедших дней + прогноз = среднийДень × всегоДней.
- **Сравнение с прошлым месяцем:** тот же факт-расчёт для месяца перед `ym` → дельты по доходу, расходу, среднему дню.
- **Выводы (правила, до 3):** напр. категория > +25% к среднему по категориям → текст; доход −>10% к плану; нетто<0 → «в минус на X»; расход +>15% к прошлому месяцу; если месяц хороший — «отложил X». Каждый — `{text, tone:'good|bad|warn'}`.

*Прогресс целей по месяцам в данных не хранится — этот пункт сознательно опускаем, не выдумываем.*

## Рендер — `renderMonthlyReviewBody(r)`

Одна функция → HTML-строка, её зовут и модалка, и архив. KPI-строка (доход ф/п, расход ф/п, нетто) → таблица категорий → блок сравнения → список выводов. Имена через `esc()`. Цвета только из CSS-переменных (`--accent` норма, `--accent2` перебор/долг, `--good`, `--muted` и т.д.).

## Вкладка — `renderReviews()` (зов из `renderAll`)

- nav-кнопка после «Советы» (строка ~1525): `<button class="tab" onclick="showPage('reviews', event)">` + SVG + «Итоги».
- контейнер `<div id="page-reviews" class="page">` после закрытия `page-tips` (строка ~2450), внутри `#reviews-content`.
- `renderReviews()`: верх — `computeMonthlyReview(currentMonthKey(), {live:true})`; ниже — `Object.keys(monthlyReviews)` по убыванию, каждый через `renderMonthlyReviewBody`.

## Авто-модалка — `maybeShowMonthlyReview()` (зов в init после renderAll, рядом с pending)

- `prevMonth` = предыдущий месяц.
- если в `prevMonth` есть транзакции **и** `lastReviewSeen !== prevMonth` → посчитать, записать в `monthlyReviews[prevMonth]`, выставить `lastReviewSeen=prevMonth`, `save()`, показать модалку.
- модалка — по образцу `pending-events-modal` (динамический host, id `monthly-review-modal`, закрытие = remove()). Внутри — `renderMonthlyReviewBody` + кнопка «Понятно».
- первый запуск / нет данных за прошлый месяц → молча пропустить.

## Деплой

Поднять `CACHE_VERSION` в `service-worker.js` (`fintrack-v19` → `v20`), иначе PWA на iPhone не подтянет.

## Инварианты, которые соблюдаем

- Все суммы из `transactions[].amount` уже в ₽ — не конвертируем повторно.
- Переводы `🔄 Между своими` исключаем везде в новом коде.
- Новые поля state — только с миграцией, ничего не теряем.
- Цвета — только CSS-переменные. Имена — через `esc()`.
- Тон сообщений прямой: «в минус на X», а не «обратите внимание».
